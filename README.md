<strong>MangosZero - Vanilla WoW server</strong></h1>
<p dir="auto"><strong>Mangos</strong> is an open source project written in <a href="http://www.cppreference.com/" title="C / C++ reference" rel="nofollow">C++</a> . It's fast, runs on multiple
platforms and stores game data in <a href="https://dev.mysql.com/downloads/" title="MySQL - The world's most popular open source database" rel="nofollow">MySQL</a> or <a href="https://mariadb.org/download/" title="MariaDB - An enhanced, drop-in replacement for MySQL" rel="nofollow">MariaDB</a>. It also has
optional support for SOAP.</p>
<p dir="auto">If you liked the original incarnation of <a href="https://worldofwarcraft.com/" title="World of Warcraft" rel="nofollow">World of Warcraft</a> and still want to play it,
this is the branch for you. We provide an authentication server where you can manage your users,
and a world server which serves game content just like the original did back then.</p>
<p dir="auto">It aims to be 100% compatible with the 3 final versions of Vanilla <a href="https://worldofwarcraft.com/" title="World of Warcraft" rel="nofollow">World of Warcraft</a>,
namely <a href="http://www.wowpedia.org/Patch_1.12.1" title="Vanilla WoW - Patch 1.12.1 release notes" rel="nofollow">patch 1.12.1</a>, <a href="http://www.wowpedia.org/Patch_1.12.2" title="Vanilla WoW - Patch 1.12.2 release notes" rel="nofollow">patch 1.12.2</a> &amp; <a href="http://www.wowpedia.org/Patch_1.12.3" title="Vanilla WoW - Patch 1.12.3 release notes" rel="nofollow">patch 1.12.3</a>.
<br><strong>IT DOES NOT SUPPORT 1.13.x</strong> and beyond which is the newly released Classic Experience (NuClassic).</p>
<p dir="auto">On top of that each update is automatically built by <a href="https://github.com/mangoszero/server/actions/workflows/core_build.yml" title="Github Actions - Linux/MAC build status">Github Actions</a> (Linux/MAC) and <a href="https://ci.appveyor.com/project/MaNGOS/server-9fytl/history" title="AppVeyor Scan - Windows build status" rel="nofollow">AppVeyor</a> (Windows)
as you can see by the images in the heading above! We do love green builds, and working things.</p>
<h2 dir="auto"><a id="user-content-requirements" class="anchor" aria-hidden="true" href="#requirements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Requirements</h2>
<p dir="auto">The server supports a wide range of operating systems, and various compiler platforms.
In order to do that, we use various free cross-platform libraries and use <a href="http://www.cmake.org/" title="CMake - Cross Platform Make" rel="nofollow">CMake</a> (v3.12+) to provide
a cross-platform build system which adapts to your chosen operating system and compiler.</p>
<h2 dir="auto"><a id="user-content-operating-systems" class="anchor" aria-hidden="true" href="#operating-systems"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Operating systems</h2>
<p dir="auto">Currently we support running the server on the following operating systems:</p>
<ul dir="auto">
<li><strong><a href="http://windows.microsoft.com/" title="Microsoft Windows" rel="nofollow">Windows</a></strong>, 32 bit and 64 bit. Windows Server 2008 (or newer) or Windows 8 (or newer) is recommended.</li>
<li><strong>Linux</strong>, 32 bit and 64 bit. <a href="http://www.debian.org/" title="Debian - The Universal Operating System" rel="nofollow">Debian 7</a> and <a href="http://www.ubuntu.com/" title="Ubuntu - The world's most popular free OS" rel="nofollow">Ubuntu 12.04 LTS</a> are
recommended. Other distributions with similar package versions will work, too.</li>
<li><strong>BSD</strong>, 32 bit and 64 bit. <a href="http://www.freebsd.org/" title="FreeBSD - The Power To Serve" rel="nofollow">FreeBSD</a>, <a href="http://www.netbsd.org/" title="NetBSD - The NetBSD Project" rel="nofollow">NetBSD</a>, <a href="http://www.openbsd.org/" title="OpenBSD - Free, functional and secure" rel="nofollow">OpenBSD</a> are recommended.</li>
</ul>
<p dir="auto">Of course, newer versions should work, too. In the case of Windows, matching
server versions will work, too.</p>
<h2 dir="auto"><a id="user-content-compilers" class="anchor" aria-hidden="true" href="#compilers"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Compilers</h2>

<ul dir="auto">

<p dir="auto"><strong><a href="https://visualstudio.microsoft.com/vs/older-downloads/" title="Visual Studio Downloads" rel="nofollow">Microsoft Visual Studio</a> 32 bit and 64 bit.</strong> All editions of Visual Studio

</li>

<h2 dir="auto"><a id="user-content-dependencies" class="anchor" aria-hidden="true" href="#dependencies"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Dependencies</h2>
<p dir="auto">The server stands on the shoulders of several well-known Open Source libraries plus
a few awesome, but less known libraries to prevent us from inventing the wheel again.</p>
<p dir="auto"><strong>Please note that Linux and Mac OS X users should install packages using
their systems package management instead of source packages.</strong></p>
<ul dir="auto">
<li><strong><a href="http://git-scm.com/" title="Git - Distributed version control system" rel="nofollow">Git</a> / <a href="http://windows.github.com/" title="github - windows client">Github for Windows</a></strong>: This version control software allows you to get the source files in the first place.</li>
<li><strong><a href="https://dev.mysql.com/downloads/" title="MySQL - The world's most popular open source database" rel="nofollow">MySQL</a></strong> / <strong><a href="https://mariadb.org/download/" title="MariaDB - An enhanced, drop-in replacement for MySQL" rel="nofollow">MariaDB</a></strong>: These databases are used to store content and user data.</li>
<li><strong><a href="http://www.dre.vanderbilt.edu/~schmidt/ACE.html" title="ACE - The ADAPTIVE Communication Environment" rel="nofollow">ACE</a></strong>: aka Adaptive Communication Environment, provides us with a solid cross-platform framework for abstracting operating system specific details.</li>
<li><strong><a href="http://github.com/memononen/recastnavigation" title="Recast - Navigation-mesh Toolset for Games">Recast</a></strong>: In order to create navigation data from the client's map files, Recast is used to do the dirty work. It provides functions for rendering, pathing, etc.</li>
<li><strong><a href="http://sourceforge.net/projects/g3d/" title="G3D - G3D Innovation Engine" rel="nofollow">G3D</a></strong>: This engine provides the basic framework for handling 3D data and is used to handle basic map data.</li>
<li><strong><a href="http://zezula.net/en/mpq/stormlib.html" title="Stormlib - A library for reading data from MPQ archives" rel="nofollow">Stormlib</a></strong>: Provides an abstraction layer for reading from the client's data files.</li>
<li><strong><a href="http://www.zlib.net/" title="Zlib" rel="nofollow">Zlib</a>/<a href="http://gnuwin32.sourceforge.net/packages/zlib.htm" title="Zlib for Windows" rel="nofollow">Zlib for Windows</a></strong> provides compression algorithms used in both MPQ archive handling and the client/server protocol.</li>
<li><strong><a href="http://www.bzip.org/" title="Bzip2" rel="nofollow">Bzip2</a>/<a href="http://gnuwin32.sourceforge.net/packages/bzip2.htm" title="Bzip2 for Windows" rel="nofollow">Bzip2 for Windows</a></strong> provides compression algorithms used in MPQ archives.</li>
<li><strong><a href="http://www.openssl.org/" title="OpenSSL - The Open Source toolkit for SSL/TLS" rel="nofollow">OpenSSL</a>/<a href="http://slproweb.com/products/Win32OpenSSL.html" title="OpenSSL for Windows" rel="nofollow">OpenSSL for Windows</a></strong> provides encryption algorithms used when authenticating clients.</li>
